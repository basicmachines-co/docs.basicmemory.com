Cursor supports remote Streamable HTTP MCP servers and OAuth, so it can connect directly to Basic Memory Cloud.

## Connect Basic Memory

Open Cursor's MCP settings and add a custom server. Cursor stores MCP configuration in `mcp.json`.

Add:

```json
{
  "mcpServers": {
    "basic-memory": {
      "url": "https://cloud.basicmemory.com/mcp"
    }
  }
}
```

Use the authorization control shown for Basic Memory and complete the browser sign-in flow.

::note
Use Cursor's user-level MCP configuration to make Basic Memory available in every project. Use a project-level `.cursor/mcp.json` only when the connection should be defined by that repository.
::

## Verify the connection

Open Cursor's MCP settings and confirm that Basic Memory is enabled and its tools are available. Then ask:

```text
List my Basic Memory projects.
```

## Try it

```text
Create a note documenting this repository's authentication architecture.
```

```text
Search my notes for previous database migration decisions.
```

```text
Update our project note with the changes made in this session.
```

## Troubleshooting

**The server does not appear:** Check that `mcp.json` contains valid JSON and that the URL is exactly `https://cloud.basicmemory.com/mcp`.

**Tools are missing:** Enable Basic Memory in Cursor's MCP settings, then restart Cursor if the configuration was just added.

**Authorization failed:** Disconnect or remove the server, add it again, and repeat the OAuth flow.
