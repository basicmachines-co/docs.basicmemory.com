Snapshots preserve the state of an entire Basic Memory Cloud workspace at a particular moment. Use them before a large import, reorganization, restore, or other change you may want to undo.

Basic Memory creates automatic snapshots daily. You can also create a manual snapshot whenever you need a specific rollback point.

::tip
Trying to recover something now? See [Recover Notes](/cloud/restore-lost-content). For changes to one existing note, [File History](/cloud/file-history) is usually faster.
::

---

## What snapshots contain

A snapshot covers the projects and files in the selected workspace. From a snapshot, you can:

- Browse files as they existed at that time
- Download the snapshot as a ZIP archive
- Restore selected files or folders
- Recover a deleted project's files

A restore writes the selected historical files back to their original paths. It does not roll the entire workspace backward or remove unrelated newer files.

---

## Manage snapshots in the web app

Open **Settings → Snapshots**. The page shows the selected workspace, snapshot totals, and whether each snapshot is automatic or manual.

Snapshots require an active Cloud subscription. In a team workspace, the workspace owner manages snapshots.

Each snapshot has three actions:

- **Restore** — browse and restore selected files or folders
- **Download** — download the snapshot as a ZIP archive
- **Delete** — remove the snapshot from the list

![The current Snapshots settings page, showing workspace totals and Restore, Download, and Delete actions.](/screenshots/cloud-app/snapshots-list.png)

### Create a manual snapshot

1. Select **Create Snapshot**.
2. Enter an optional description, such as `Before importing research archive`.
3. Select **Create**.

If you leave the description blank, Basic Memory uses the current timestamp.

![The Create Snapshot dialog with an optional description field.](/screenshots/cloud-app/create-snapshot.png)

### Restore files or folders

1. Select **Restore** beside the snapshot you want to use.
2. Choose a project under **Scope**, or browse all projects.
3. Search or browse for the files and folders you need.
4. Select one or more paths.
5. Select **Restore**.

![The current snapshot restore screen with project scope, search, and selectable files and folders.](/screenshots/cloud-app/restore-files.png)

::warning
Restoring overwrites current files at the selected paths. Create a new snapshot first if you may need to return to the current versions.
::

## Use snapshots from the CLI

The CLI uses the same Cloud snapshots. Sign in first:

```bash
bm cloud login
```

### Create and list snapshots

```bash
bm cloud snapshot create "Before reorganization"
bm cloud snapshot list
bm cloud snapshot list --limit 20
```

### Inspect a snapshot

Show its metadata:

```bash
bm cloud snapshot show <snapshot-id>
```

Browse all files or filter by a path prefix:

```bash
bm cloud snapshot browse <snapshot-id>
bm cloud snapshot browse <snapshot-id> --prefix research/
```

### Restore a file or folder

Restore uses `bm cloud restore`, not the `snapshot` command group:

```bash
bm cloud restore research/important.md --snapshot <snapshot-id>
bm cloud restore research/ --snapshot <snapshot-id>
```

The CLI previews the affected files and asks for confirmation. Add `--force` only when you intentionally want to skip that prompt.

### Delete a snapshot

```bash
bm cloud snapshot delete <snapshot-id>
```

The CLI asks for confirmation unless you add `--force`.

## When to create a manual snapshot

Create one before:

- Importing a large collection
- Moving or renaming many files
- Reorganizing projects or folders
- Running a bulk edit or automation
- Restoring files from an older snapshot
- Deleting substantial content

Use a description that explains the change you are about to make. The automatic daily snapshots remain useful for general recovery, while a named manual snapshot gives you a precise rollback point.

## Snapshots and file history

| Use | Best option |
|-----|-------------|
| Undo or compare edits to one existing note | [File History](/cloud/file-history) |
| Recover a deleted note | Snapshot |
| Recover a folder or deleted project | Snapshot |
| Preserve the workspace before a risky bulk change | Manual snapshot |
| Keep an offline copy of a point in time | Download snapshot |
