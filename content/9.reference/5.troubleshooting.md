Run these before digging deeper:


```bash

```

If you are using Cloud from the CLI:

```bash
bm cloud status
```

## Cloud MCP connection

Cloud MCP uses the hosted endpoint:

```text
https://cloud.basicmemory.com/mcp
```

Use this URL when a client asks for a remote MCP server, custom connector, custom app, or MCP endpoint.

### The AI tool cannot connect

Check:

1. The endpoint is exactly `https://cloud.basicmemory.com/mcp`.
2. You completed the Basic Memory authorization flow.
3. You are signed in to the same Basic Memory account that owns or has access to the workspace.
4. Your client supports remote MCP or custom MCP connectors.
5. The connector/app is enabled in the current chat or project, if the client requires per-chat enablement.

Then remove and re-add the connector from the AI client.

### The tool connects but cannot see projects

Ask the assistant:

```text
List my Basic Memory projects.
```

If it returns no projects:

- Open [app.basicmemory.com](https://app.basicmemory.com) and confirm you can see the workspace and projects there.
- If you were invited to a team workspace, confirm you accepted the invitation with the invited email address.
- Switch to the correct workspace in the Basic Memory web app.
- Re-authorize the MCP connector so the client refreshes access.

### Claude does not use Basic Memory

Claude custom connectors may need to be enabled in each conversation.

Open the **+** menu beside the message box, choose **Connectors**, and enable Basic Memory. On managed Claude plans, an organization owner may need to approve the connector first.

See [Claude](/integrations/claude-desktop).

### ChatGPT does not use Basic Memory

ChatGPT custom apps may need to be invited into a chat from the message composer or tools menu. If developer mode or custom apps are unavailable, your ChatGPT plan, workspace role, or administrator policy may not allow them.

See [ChatGPT](/integrations/chatgpt).

### Codex, Cursor, VS Code, or Gemini cannot connect

Check the integration page for your client:

- [Codex](/integrations/codex)
- [Cursor](/integrations/cursor)
- [VS Code](/integrations/vscode)
- [Gemini CLI](/integrations/gemini)

The durable rule: use the Cloud endpoint for remote MCP, and use `uvx basic-memory mcp` only for local MCP.

## Cloud app issues

### You are in the wrong workspace

Use the workspace switcher in the Basic Memory web app. Personal, organization, and invited workspaces are separate. Projects and permissions are scoped to the selected workspace.

If you expected a team project:

- Confirm the invite was accepted.
- Confirm you are signed in with the invited email.
- Ask an owner to check your role and project access in **Settings → Teams**.

### Subscription or seat problem

Workspace access depends on the subscription and seat allocation.

- Owners manage seats and billing in **Settings → Billing**.
- Pending invitations do not use a seat until accepted.
- Active members use seats.
- Deactivated members lose access and free a seat.

See [Share Basic Memory With Your Team](/teams/about).

### API key authentication failed

API keys are useful for CLI, automation, and MCP clients that support bearer tokens.

If a key stops working:

1. Check whether it was revoked in **Settings → API Keys**.
2. Create a new key if needed.
3. Update the environment variable, CLI config, or client header that uses the key.

For CLI routing:

```bash
bm cloud api-key save bmc_your_key_here
bm cloud status
```

See [API Keys](/cloud/api-keys).

## Local install and MCP

Local mode uses the open-source CLI and a local MCP server. Your source of truth is a Markdown folder on your computer.

### `bm` is not found

Install or reinstall Basic Memory:

::code-group
```bash [uv]
uv tool install basic-memory
```

```bash [Homebrew]
brew tap basicmachines-co/basic-memory
brew install basic-memory
```
::

Then open a new terminal and run:

```bash
bm --version
```

See [Install locally](/local/local-install).

### `uvx` is not found by an MCP client

GUI apps often do not inherit your shell `PATH`.

Find the absolute path:

```bash
which uvx
```

Then use that full path as the MCP `command`:

```json
{
  "mcpServers": {
    "basic-memory": {
      "command": "/Users/you/.local/bin/uvx",
      "args": ["basic-memory", "mcp"]
    }
  }
}
```

Restart the MCP client after changing the config.

### Local MCP tools are missing

Check:

1. The client config uses `uvx basic-memory mcp` or `bm mcp`.
2. The JSON/TOML syntax is valid.
3. You restarted the client after editing the config.
4. `bm --version` works in a terminal.
5. `bm project list` shows at least one project.

You can start the MCP server manually for a quick sanity check:

```bash
bm mcp
```

Stop it with `Ctrl+C` after confirming it starts.

### Local notes are in the wrong place

List projects:

```bash
bm project list
```

The default local project is usually `~/basic-memory`. Add or switch projects when needed:

```bash
bm project add research ~/Documents/research
bm project default research
```

### File changes are not appearing

Check:

```bash
bm status --verbose
bm doctor
```

Then verify:

- The file is inside a configured project folder.
- The file is Markdown if you expect it to become a note.
- The file is not excluded by `.gitignore` or `.bmignore`.
- Basic Memory can read and write the folder.

If the local index is inconsistent, rebuild it:

```bash
bm reset --reindex
```

::warning
`bm reset --reindex` rebuilds the local database from files. It does not delete Markdown files, but it can take time for large projects.
::

## Sync and routing

Cloud Sync and local/cloud routing are advanced workflows. You do not need them to use Basic Memory Cloud in the web app or through hosted MCP.

### Cloud-routed CLI commands fail

Re-authenticate:

```bash
bm cloud logout
bm cloud login
bm cloud status
```

For automation or hybrid setups, use an API key:

```bash
bm cloud api-key save bmc_your_key_here
bm cloud status
```

### A project should be local but routes to Cloud

Switch it back:

```bash
bm project set-local research
bm cloud status
```

### A project should route to Cloud but stays local

Save Cloud credentials and route the project:

```bash
bm cloud api-key save bmc_your_key_here
bm project set-cloud research
bm cloud status
```

See [Local & Cloud Routing](/cloud/routing).

### Cloud Sync says the project has no local sync path

Attach a local directory:

```bash
bm cloud sync-setup research ~/Documents/research
```

Then establish or check sync according to the workflow:

```bash
bm cloud bisync --name research --resync --dry-run
bm cloud bisync --name research --resync
```

See [Cloud Sync](/cloud/cloud-sync).

### First bisync reports an empty directory

Add a starter Markdown file before establishing the first baseline:

```bash
printf '# Research\n' > ~/Documents/research/README.md
bm cloud bisync --name research --resync
```

### Bisync reports too many deletes

Stop and inspect the dry run:

```bash
bm cloud bisync --name research --dry-run
```

Do not run a new `--resync` baseline until you understand why files would be removed. If this is a large reorganization, create a [Cloud Snapshot](/cloud/cloud-snapshots) first.

## Search and content

### The assistant cannot find a note

Try a direct search:

```bash
bm tool search-notes --query "the topic you expect"
```

Then try reading by title, path, or permalink:

```bash
bm tool read-note "My Note"
bm tool read-note "folder/my-note.md"
```

If you are using an AI tool, ask it to list projects first. Many "missing note" problems are project-selection problems.

### Search results are too broad

Use a more specific search type or filters:

```bash
bm tool search-notes --query "authentication architecture" --search-type hybrid
bm tool search-notes --query "login flow" --search-type vector
bm tool search-notes --query "tag:security"
```

Use [metadata search](/concepts/metadata-search) when frontmatter fields matter.

### A note exists but relations or observations do not appear

Check the Markdown format:

```markdown
## Observations
- [decision] Use Cloud as the default setup path

## Relations
- part_of [[Docs Overhaul]]
```

Then run:

```bash
bm status
bm doctor
```

See [Observations and Relations](/concepts/observations-and-relations).

### Memory URLs do not resolve

Check the title, path, or permalink:

```text
memory://my-note
memory://folder/my-note
```

If a note was renamed or moved, search for it by title first, then use the current permalink. See [Memory URLs](/concepts/memory-urls).

## Performance

### Commands are slow

Check project size and status:

```bash
bm project info main
bm status
```

For large local projects:

- Archive generated or inactive files outside the project folder.
- Exclude dependency/build folders with `.gitignore` or `.bmignore`.
- Keep huge binary files outside the project unless you need them available through `read_content`.
- Use smaller `page_size` values in MCP/search calls.

### Semantic search takes time after adding many notes

Embedding and indexing work can take time for large imports. Let indexing finish, then check:

```bash
bm status
bm doctor
```

## Configuration

### Config changes are not taking effect

Restart the MCP client after editing config files. For long-running local servers, stop and restart `bm mcp`.

### Config file has invalid JSON

Validate it:

```bash
python -m json.tool ~/.basic-memory/config.json
```

Common issues:

- Missing commas between properties
- Trailing commas after the final property
- Unquoted strings

See [Configuration](/reference/configuration).


### Logs

Basic Memory writes local logs under `~/.basic-memory/`. If you ask for help, include:

- What you expected to happen
- What happened instead
- The command or client you used
- The relevant error message
- Your operating system
- Whether the project is Cloud, local, or hybrid

Do not paste API keys, OAuth tokens, private notes, invitation links, or billing information into public support channels. Use [Contact Support](/reference/contact-support) for private issues.
