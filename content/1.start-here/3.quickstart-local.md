title: "Quickstart: Open Source"
description: Install Basic Memory locally, connect an MCP client, and create your first note.
---

The open-source version runs on your computer. Your source of truth is a folder of Markdown files you control.

## 1. Install Basic Memory

We recommend installing with `uv`:

::code-group
```bash [uv]
uv tool install basic-memory
```

```bash [Homebrew]
brew tap basicmachines-co/basic-memory
brew install basic-memory
```
::

Confirm that the CLI is available:

```bash
bm --version
```

## 2. Connect an MCP client

Configure your AI tool to launch the local MCP server:

```json
{
  "mcpServers": {
    "basic-memory": {
      "command": "uvx",
      "args": ["basic-memory", "mcp"]
    }
  }
}
```

Restart the AI tool after changing its configuration.

::note
The install command uses `uv tool install` so you can run `bm` in your terminal. The MCP snippet uses `uvx` so your AI client can launch Basic Memory directly. If the client cannot find `uvx`, run `which uvx` and use the returned absolute path as `command`.
::

Client-specific instructions are available under [Connect AI tools](/integrations?deployment=local).

## 3. Create your first note

Ask your connected AI tool:

```text
Create a Basic Memory note called "Getting Started" that explains what I want
to keep in this knowledge base.
```

Then ask:

```text
Read my "Getting Started" note.
```

This verifies both write and read access.

## 4. Find the Markdown files

The default `main` project lives at `~/basic-memory`. Open that folder in any Markdown editor, file browser, Obsidian vault, or Git repository.

To use another folder:

```bash
bm project add research ~/Documents/research
bm project default research
```

Basic Memory indexes Markdown from that folder without taking ownership of the files.

## 5. Try the CLI

```bash
bm project list
bm status
bm tool search-notes --query "getting started"
```

The CLI and MCP server use the same projects and index.
