Basic Memory Cloud gives you a hosted workspace, a web app, and a remote MCP endpoint your AI tools can use from anywhere. You can start with Claude, Codex, ChatGPT, or another MCP-compatible client.

This guide follows the same path as the in-app onboarding. It uses Claude as the example client because that is the default walkthrough in the product, but the same endpoint works with Codex and other MCP tools.

```text
https://cloud.basicmemory.com/mcp
```

## 1. Create your account

Go to [app.basicmemory.com](https://app.basicmemory.com) and create an account.

You can sign up with email or a supported identity provider such as Google, Microsoft, or GitHub.

After email verification, new workspace owners choose a plan, name their workspace, and complete checkout. Invited users follow the invitation flow for the workspace they were invited to.

---

## 2. Choose an AI tool

After checkout or invite acceptance, Basic Memory opens the **Getting Started** onboarding screen.

The first step asks which AI tool you want to connect:

- Claude
- Codex
- ChatGPT
- Other MCP tools

Pick the assistant you use most. You can connect more tools later.

::tip
Need client-specific details? Use the integration guides:

- [Claude](/integrations/claude-desktop)
- [Codex](/integrations/codex)
- [ChatGPT](/integrations/chatgpt)
::

---

## 3. Copy your MCP endpoint

The onboarding screen shows your remote MCP endpoint:

```text
https://cloud.basicmemory.com/mcp
```

Paste that URL wherever your AI tool asks for a remote MCP server, connector, or custom MCP server.

Use the setup guide for your client:

- [Claude](/integrations/claude-desktop)
- [Claude Code](/integrations/claude-code)
- [ChatGPT](/integrations/chatgpt)
- [Gemini CLI](/integrations/gemini)
- [Codex](/integrations/codex)
- [Cursor](/integrations/cursor)

These guides cover where to enter the endpoint, how to authorize Basic Memory, and how to verify the connection.

::tip
The setup guide remains available later from **Settings -> MCP** in the Basic Memory web app.
::

---

## 4. Create your first memory

Once your AI tool is connected, send the prompt from the onboarding screen:

```text
Help me get started with Basic Memory. First, explain in a few bullets how you can use Basic Memory with me to remember project context, decisions, and next steps. Then create a note titled "Getting Started with Basic Memory" that includes:
- What Basic Memory is for
- How I should use it with you
- One simple thing I should try next
```

This confirms that your AI tool can write to your Basic Memory workspace.

If the prompt succeeds, your assistant creates a note in your cloud workspace. You can open it in the web app and continue editing it there.

::tip
Want your assistant to use Basic Memory more consistently? After you finish the quickstart, add the recommended [Basic Memory instructions](/start-here/getting-started#teach-your-ai-when-to-use-basic-memory) to your AI tool.
::

---

## 5. Invite your team

Standalone workspaces include a final onboarding step for collaboration.

You can invite teammates immediately from **Team Settings**, or skip this and invite people later from **Settings -> Teams**.

::note
Invited users may not see this step. If you joined an existing workspace, your onboarding focuses on connecting an AI tool and creating your first memory.
::

---

## 6. Open the Getting Started project

After onboarding, Basic Memory opens the **Getting Started** project. It includes eight tutorial notes that explain:

- What Basic Memory is
- The toolset your AI assistant can use
- How Basic Memory structures knowledge
- Markdown, imports, sync, and next steps

Your workspace also includes a **Main** project for your own notes.

![The Basic Memory web app open to the Getting Started project, with eight tutorial notes and the welcome note selected.](/screenshots/cloud-app/getting-started-project.png)

---

## What to try next

Use these prompts with your connected assistant:

| Prompt | What it does |
|--------|--------------|
| `Create a note about what we just discussed.` | Writes a new note to your workspace. |
| `Find my notes about [topic].` | Searches your knowledge base. |
| `What decisions have we made about [project]?` | Uses your notes as context. |
| `Add this to my notes about [topic].` | Updates existing knowledge. |
| `Show me recent notes.` | Lists recent workspace activity. |
